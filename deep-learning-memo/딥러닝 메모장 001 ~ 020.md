Stratified Group K-Fold는 기존 K-Fold와 어떻게 다른가?

알반 K-Fold: 데이터를 그냥 순서대로 등분합니다.

Stratified K-Fold: 데이터셋 전체의 [정상:폐렴] 비율이 1:3이라면, 
쪼개진 5개의 각 Fold에도 정밀하게 1:3 비율을 유지하도록 분할합니다.

Stratified Group K-Fold: 클래스 비율(1:3)도 정확히 맞추면서, 
동시에 "같은 환자 ID를 가진 사람들은 무조건 하나의 Fold에만 통째로 들어가라"고 제약을 거는 방식입니다.

이렇게 하는 이유: 한 환자가 찍은 엑스레이 3장이 각각 Fold0, 1, 2로 쪼개져서 들어가면, 
모델은 폐렴의 증상을 배우는 게 아니라, 그 환자의 뼈 구조나 사진의 고유한 밝기를 외워버립니다.

합성곱 신경망 도입 배경

기존의 신경망(다층 퍼셉트론, MLP)이 가진 데이터 표현의 한계를 극복하기 위해서입니다.
nn.Linear 대신 nn.Conv2d를 사용하는 이유는 크게 세. 가지 이유가 있습니다.
1. 공간 정보의 손실 방지
일반적인 신경망에 이미지를 넣으려면 2차원 이미지를 1차원 배열로 길게 펼쳐야 합니다.(Flatten)
문제점: 이미지는 인접한 픽셀끼리 밀접한 관계를 갖습니다. 이미지를 1차원으로 펼치는 순간 정보가 완전히 깨집니다.
해결: 합성곱은 커널(필터)를 사용해 2차원 구조를 그대로 유지하며 훑기 때문에, 이미지 내의 국소적 특징을 효과적으로 포착합니다.

2. 파라미터 폭발 문제
이미지 해상도가 조금만 높아져도 일반 신경망은 학습해야 할 가중치(weight)가 기하급수적으로 늘어납니다.
해결: 합성곱은 가중치 공유(weight share) 방식을 사용합니다.

3. 변형에 대한 저항성
이미지 속의 물체가 조금만 옆으로 이동해도 일반 신경망은 이를 전혀 다른 데이터로 인식하는 경우가 많습니다.

역전파(backpropagation)란?
역전파 알고리즘은 출력값에 대한 입력값의 기울기를 출력층 layer에서부터 계산하여 거꾸로 전파시키는 것이다.
이렇게 거꾸로 전파시켜서 최종적으로 출력층에서의 output값에 대한 입력층에서의 input data의 기울기 값을
구할 수 있다. 이 과정에서 중요한 개념인 chain rule이 사용된다.
 출력층 바로 전 layer에서부터 기울기를 계산하고 이를 점점 거꾸로 전파시키면서
 전 layer들에서의 기울기와 서로 곱하는 형식으로 나아가면 최종적으로 
 출력층의 output에 대한 입력층에서의 input의 기울기 값을 구할 수 있다.

신경망을 통과하면서 만들어진 굉장히 복잡한 고차원의 합성함수의 기울기를 계산하기 위해서는 
미분의 연쇄법칙을 활용해야 한다.
합성함수의 기울기를 계산하려면 이전의 함수의 미분값이 필요하고, 그리고 이전의 함수의 미분값을
구하기 위해서는 그 이전의 미분값이 필요하다.
이런식으로 이전의 값이 다음에 구하는 값에 필요하게 된다.
이것이 뒤에서 앞으로 전파하는 역전파라는 말이 나온 이유다.
```

class Model(nn.Module):
    def __init__(self):
        super().__init__()
        self.layer1 = nn.Sequential(
            nn.Conv2d(3, 64, 5),
            nn.ReLU(inplace=True),
            nn.MaxPool2d(2)
        )
        self.layer2 = nn.Sequential(
            nn.Conv2d(64, 30, 5),
            nn.ReLU(inplace=True),
            nn.MaxPool2d(2)
        )
        self.layer3 = nn.Sequential(
            nn.Linear(30*5*5, 10, bias=True),
            nn.ReLU(inplace=True)
        )
    def forward(self, x):

        x = self.layer2(x)
        x = x.view(x.shape[0], -1)
        x = self.layer3(x)
        return x

```
nn.Linear(5, 3)일 때, Weight는 (3, 5)행렬, Bias는 (3,) 벡터가 되는 이유
1. Weight가 (3, 5)인 이유: 행렬 곱셈의 규칙
nn.Linear(5, 3)은 5개의 입력을 받아 3개의 출력을 내보낸다.
입력(x): 보통 (1, 5) 형태의 행벡터로 들어옵니다.
연산: 입력 x에 가중치 W를 곱해서 출력을 만들어야 합니다.
규칙: (1, 5) 행렬 뒤에 어떤 행렬을 곱해서 (1, 3)이라는 결과를 얻으려면 가중치 행렬은 반드시 (5, 3) 형태여야 합니다.
근데.. 왜 PyTorch는 (3, 5)라고 할까요??
PyTorch는 내부적으로 가중치를 (출력차원, 입력차원) 형태로 저장합니다. 
그리고 실제 계산할때는 전치(Transpose)하여 계산합니다.
이는 수식의 편의성과 효율성을 위한 딥러닝 프레임워크들의 약속입니다.

2. Bias가 (3,)인 이유: 출력 노드와의 1:1 대응
출력 개수: 우리가 설정한 out_features가 3이므로, 결과물은 3개의 숫자가 나옵니다.
