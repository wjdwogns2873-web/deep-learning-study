ET는.. ElementTree야. 
ET.parse()를 하면 tree에는 어떤 값이 담기는거야? 
그리고.. tree.getroot()는? 디렉토리명을 가져오는건가?

1. tree = ET.parse(xml_path)
담기는 값: ElementTree 객체 (전체 xml 문서 구조)
설명: xml 파일 전체를 읽어와서 메모리에 트리 구조로 올린 '문서 전체 조절기'

2. root = tree.getroot()
담기는 값: Element 객체 (xml의 가장 첫 번째 최상위 태그)
설명: xml 파일의 중심 줄기가 되는 최상위 루트 태그를 가져옵니다.

예)
xml

<?xml version="1.0"?>
<company name="네이버">
	<department>개발부</department>
	<department>인사부</department>
</company>

ET.parse(xml_path): xml 문서 전체 구조를 잡고 있는 객체
tree.getroot(): 최상위 태그인 <company> 태그

질문1. 
tuple(zip(*batch))
이것에 대해 설명해줄래? tuple은 당연히 튜플 형태일 것인데.. zip(*) 이게 뭘지..

	데이터로더가 배치 사이즈를 2로 설정해 데이터를 긁어오면, 
	batch라는 변수 안에는 아래와 같이 가로로 묶인 리스트가 들어오게 됩니다.
	
	<파이썬 코드>
	batch = [ 
		(이미지1, 정답DTO1),  # 0번째 아이템 
		(이미지2, 정답DTO2)  # 1번째 아이템 
	]

	하지만 파이토치 모델에 데이터를 입력할 때는 [이미지1, 이미지2] 묶음과 
	[정답DTO1, 정답DTO2] 묶음으로 세로로 분리된 구조가 필요합니다. 
	그래서 zip(*batch)를 사용합니다.
	(1) *batch -> (이미지1, 정답DTO1), (이미지2, 정답DTO2)
	(2) zip(...) -> 분리된 녀석들의 첫번째 원소끼리 엮고, 두번째 원소끼리 세로로 엮습니다.
	(3) tuple(...) -> 그 결과를 수정 불가능한 튜플로 감싸서 아래와 같은 구조를 만듭니다.
	
	(
		(이미지1, 이미지2), 
		(정답DTO1, 정답DTO2)
	)
	
	이렇게 함으로써 학습 루프에서 images에는 이미지들, targets에는 정답DTO들만 분리되어 들어갑니다.

질문2.

예전에는 그냥 바로 model.parameters()를 썼는데
이번에는 requires_grad가 true인 parameters만 가져오는 거야? 원래 이렇게 하는게 원칙인 건가?

	전이 학습(Transfer Learning)을 할 때는 가중치 고정 레이어를 옵티마이저 관리 대상에서 제외하여
	메모리와 연산속도를 아끼기 위한 테크닉입니다.
	이전 이미지 분류 할 때는 모델을 처음부터 새로 만들었거나 전체 레이어를 다 학습시켰기 때문에
	그냥 model.parameters()를 옵티마이저에 통째로 넘겨줘도 무방했습니다.
	
	하지만 이번에 사용한 Faster R-CNN은 이미 수백만 장의 이미지로 학습된 거대한 뼈대를 품고 있습니다.
	진짜 미분을 계산해서 가중치를 업데이트할(requires_grad=True) 핵심 멤버들만 리스트로 추립니다.

targets = [{k: v.to(device) for k, v in t.items()} for t in targets]
