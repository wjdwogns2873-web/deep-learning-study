```
(50000, 3072) 행렬을 어떤 기준으로 reshape(50000, 32, 32, 3) 되는건지.
3072개 숫자의 배치 기준이 무엇인가요?
reshape의 원칙: 기존 메모리에 일렬로 서 있는 순서를 절대 흐뜨리지 않고, 앞에서부터 차례대로 넣는다.
[0번째 ~ 1023번째] (총 1024개): 오직 빨간색(R) 픽셀들만 가로 세로 순서대로 쫙 서 있음
[1024번째 ~ 2047번째] (총 1024개): 오직 초록색(G) 픽셀들만 가로 세로 순서대로 쫙 서 있음
[2048번째 ~ 3071번째] (총 1024개): 오직 파란색(B) 픽셀들만 가로 세로 순서대로 쫙 서 있음

처음부터 reshape(-1, 32, 32, 3)을 한다면..
넘파이는 기계적으로 앞에서부터 숫자를 채우기 시작합니다.
우리가 원하는 진짜 이미지 한 픽셀([0, 0] 위치)은 [R, G, B] 3개가 나란히 묶여있어야 정상적인 색상이 나옵니다.
하지만 넘파이에게 맨 뒤 차원을 3으로 쪼개달라고 주문하면, 
넘파이는 일렬로 선 줄에서 맨 앞의 3개를 뚝 잘라와서 0번째 픽셀의 [R, G, B] 방에 넣어버립니다.
넘파이가 채운 0번째 픽셀: [0번째 픽셀, 1번째 픽셀, 2번째 픽셀] -> 셋 다 빨간색(R) 데이터만 들어옴
넘파이가 채운 1번째 픽셀: [3번째 픽셀, 4번째 픽셀, 5번째 픽셀] -> 셋 다 빨간색(R) 데이터만 들어옴

1단계: .reshape(-1, 3, 32, 32)
"일단 채널 3개(R, G, B)로 찢고, 그 뒤에 각각 세로 32, 세로 32짜리 모양으로 해줘."

2단계: .transpose(0, 2, 3, 1)
모양은 완벽하게 복원되었지만, 파이토치나 PIL 이미지 라이브러리는 시각화할 때 채널(3)이 맨 뒤에 붙어 있는 구조를 원합니다.

(64, 3, 32, 32)
conv1 -> (64, 32, 32, 32)
conv2 -> (64, 64, 32, 32)
pool ->  (64, 64, 16, 16)
conv3 -> (64, 128, 16, 16)
pool ->  (64, 128, 8, 8)

```
model.train()을 쓴 이유? 모델에게 "지금은 시험이 아니라 연습이야" 라고 말해주는 것
복잡한 모델로 가면 성능을 올리기 위해 훈련할 때와 평가할 때 완전히 다르게 행동해야 하는 특수 레이어들이 추가됩니다.
대표적으로 드롭아웃과 배치 정규화입니다.
1. 드롭아웃(Dropout)
훈련할 때(model.train()): 과적합을 막기 위해 뉴런들을 랜덤하게 툭툭 끊어가며 스파르타식으로 훈련합니다.
평가할 때(model.eval()): 실전 시험을 볼 때는 모든 뉴런을 총동원해서 완벽히 정답을 맞혀야 하므로, 다 켭니다.

2. 배치 정규화(Batch Normalization)
훈련할 때(model.train()): 지금 들어온 64장 짜리 미니 배치 데이터의 평균과 분산을 기준으로 데이터를 정렬합니다.
평가할 때(model.eval()): 테스트 데이터 한 장이 들어왔을 때는 비교할 대상이 없으므로, 훈련할 때 미리 구해둔 '전체 데이터의 평균과 분산'을 가져와서 얌전하게 계산합니다.

전에는 안 쓴 이유?
이전 단계에서 다뤘던 초간단 신경망들에는 드롭아웃이나 배치 정규화같은 특수 레이어가 없었습니다.
그런 레이어가 없을 때는 model.train()을 쓰든 안 쓰든 똑같이 작동합니다.
현업에서 딥러닝 엔지니어들은 "나중에 실수로 모드가 꼬여서 모델이 오작동하는 것을 막기 위해"
습관적으로 학습 루프 시작 전에 model.train()을 적어줍니다.

학습을 시작할 때는 무조건 -> model.train()
나중에 테스트 데이터로 성능 점수를 잴 때는 무조건 -> model.eval()
```

```
왜 ResNet이 그렇게 잘할까?
망이 깊어질수록 학습이 더 잘되어야 할 것 같지만, ResNet 이전에는 오히려 층이 깊어지면 성능이 떨어지는 기현상이 있었습니다. ResNet은 이 문제를 해결했습니다.
1. 기울기 소실의 구원자
레이어가 깊어지면 역전파 과정에서 미분값이 앞쪽 레이어로 전달되다가 점점 작아져
결국 0이 되어버리는 문제가 발생합니다. 앞쪽 레이어들이 학습을 전혀 못 하게 되는 것이죠.
ResNet은 이를 잔차 연결(Residual Connection / Skip Connection)로 해결했습니다.

기존 방식: 입력 x가 들어오면 복잡한 연산을 거쳐 완전히 새로운 특징인 H(x)를 뽑아내야 했습니다.
ResNet 방식: 입력 지름길(Skip Connection)을 통해 연산 결과에 그대로 더해줍니다. H(x) = F(x) + x가 됩니다.
이 간단한 덧셈이 기적이 됩니다. 미분을 할 때 지름길로 전달된 x 덕분에
최소한 '+1'이라는 기울기가 항상 확보됩니다. 아무리 층이 깊어져도 앞쪽 레이어까지 기울기가 안전하게 배달되는 구조인 겁니다.

2. "아무것도 안 하기"
딥러닝 모델에게 "입력받은 데이터를 아무 가공도 하지 말고 그대로 다음 레이어로 넘겨라"고 하면, 의외로 이것을 어려워합니다.
수많은 가중치(W)와 편향(b)을 절묘하게 0으로 맞춰야 하기 때문입니다.
하지만 ResNet은..
만약 특정 레이어가 더 이상 학습할 필요가 없다면, 내부 가중치들을 그냥 0으로 수렴시키면 됩니다.
그러면 F(x)=0이 되고, 남는 것은 x 뿐이므로 입력이 출력으로 그대로 복사 (H(x)=x)됩니다.
이를 항등 매핑(Identity Mapping)

3. 최적화하기 쉬운 잔차(Residual) 학습
ResNet이라는 이름은 잔차(Residual, 남은 것)를 학습한다는 뜻에서 왔습니다.
이미 확보된 x를 제외한 나머지 변화량(잔차)인 F(x)=H(x)-x만 학습하라고 문제를 바꾸어 주는것입니다.
이미 베이스라인(x)을 깔고 가기 때문에, 모델은 미세한 차이(잔차)만 튜닝하면 됩니다.
그래서 학습 속도가 빨라지고 훨씬 안정적으로 Loss가 떨어지게 됩니다.
